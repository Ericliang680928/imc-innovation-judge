## 📌 一、專案緣起

### 起點
- 使用者擔任 **IMC 第 21 屆創新獎評審委員**
- 透過 `/imc-innovation-judge` Skill 取得官方評審辦法
- 需要加入「**創新徵文類**」(原本 Skill 只有 4 大類組)
- 想做成自動化系統,評審投件徵文/提案

### 目標
1. 上傳投件檔案(PDF/Word/PPT)→ 自動評審
2. 模擬真實評審團(多位 AI 評審獨立打分)
3. 輸出完整評審報告(可下載)
4. 提供桌面捷徑,方便日常使用
5. 探索雲端化,讓 IMC 社友共用

## 🎯 二、已完成功能

### A. 核心評審功能
- ✅ **5 大類組評審**:
  - 🔬 研發製造類(三階段,各 40 分,合計 120)
  - 💼 服務行銷(含財金)類(三階段,各 40 分,合計 120)
  - 🚀 新創企劃類(二階段,各 50 分,合計 100)
  - 🤝 社務創新類(二階段,各 50 分,合計 100,「創益」非創利)
  - ✍️ **創新徵文類**(單階段書面,100 分,新增)

- ✅ **評審 Skill 升級**:`anthropic-skills:imc-innovation-judge` 已加入創新徵文類完整評分表

### B. 多評審委員制
- ✅ **5 種評審 Persona**:
  - 🔍 嚴格派(質疑原創、要實證,60-75 分)
  - ⚖️ 平衡派(綜合中庸,70-82 分)
  - 🌟 鼓勵派(看亮點、引導,75-88 分)
  - 💼 商業派(看可行性、財務,65-80 分)
  - ✨ 創意派(看奇想、反差,70-90 分)

- ✅ **去極值平均(IMC 官方計分法)**:
  - N=1:單一評審
  - N=3:標準(去掉最高最低,取中間 1 位)
  - N=5:完整評審團(去掉最高最低,取中間 3 位平均)

- ✅ **自動推薦組合**(依類組):
  - 創新徵文類 → 創意派、平衡派、嚴格派
  - 新創企劃類 → 商業派、平衡派、嚴格派
  - 社務創新類 → 鼓勵派、平衡派、嚴格派
  - 研發/服務 → 嚴格派、商業派、平衡派

### C. 評審輸出格式
評審報告 9 大段落:
1. **基本資訊**
2. **🚀 快速摘要**(評論 + 3-5 建議 + 3-5 問題,30 秒看完版)
3. **評分表**(逐項給分 + ≥3 句說明)
4. **💪 亮點**(≥5 項,引用文中段落)
5. **⚠️ 改善建議**(≥5 項,可執行方向)
6. **❓ 評審提問**(≥10 個,涵蓋多面向)
7. **🎯 評審總評**(≥300 字)
8. **評審建議**(🟢/🟡/🔴 晉級判斷)
9. **給參賽者的綜合提醒**

### D. AI 內容偵測
- ✅ 從 6 面向判斷 AI 生成機率:
  - 詞彙多樣性、句式結構、結構化程度

- ✅ 輸出總體 AI 機率(0-100%)+ 可疑段落 + 改善建議

## 🛠️ 七、開發歷程中遇到的問題與解決

### 問題 1:Skill 載入舊版
- **狀況**:更新 SKILL.md 後,Streamlit 載入的還是舊版快取
- **解法**:本次對話中重新呼叫 Skill,Claude Code 重新載入後正常

### 問題 2:Streamlit `source_name` NameError
- **狀況**:AI 偵測按鈕引用未定義變數
- **解法**:統一變數名 `single_name`,提前初始化

### 問題 3:multiselect 切換 N 不更新
- **狀況**:改評審人數時推薦組合沒重新帶入
- **解法**:用動態 `widget_key` + `on_click` callback

### 問題 4:Streamlit widget state 不能後改
- **狀況**:按鈕修改 session_state 觸發 API Exception
- **解法**:按鈕放在 widget 前 + 用 `on_click` callback

### 問題 5:Groq TPM 爆量
- **狀況**:長文徵文 24K tokens / 免費 TPM 6000 = 直接超 4 倍
- **解法**:診斷訊息建議切到 OpenRouter / Gemini

### 問題 6:OpenRouter 402 insufficient_quota
- **狀況**:Llama 3.3 70B 改成「需充值 $10 才能用」
- **解法**:預設改用 `nemotron-nano-9b-v2:free`(真無門檻)
- **fallback 鏈**:大模型自動降到無充值門檻組

### 問題 7:模型清單下架
- **狀況**:`deepseek-chat-v3-0324:free` 已下架,系統提取 OpenRouter 即時清單
- **解法**:模型清單全面更新為 2026 年可用版本

### 問題 8:評分超出滿分(55/50 = 110%)
- **狀況**:AI 給分超過配分上限 + `_parse_score` 抓「0–10」第一個數字 = 0
- **解法**:
  - Prompt 加「絕對紅線」與自我檢核
  - 程式 `_clamp_score` 自動截斷
  - `_parse_score` 正確處理範圍格式

### 問題 9:.bat 雙擊沒反應
- **狀況**:.bat 內中文編碼問題 + `if () (...)` 區塊內含 URL `:` 與 `"`
- **解法**:純 ASCII 改寫,改用 `goto :LABEL` 跳轉

### 問題 10:桌面捷徑沒出現
- **狀況**:OneDrive 重定向桌面到 `C:\Users\owner\OneDrive\桌面\`(中文)
- **解法**:`.bat` 改用 `[Environment]::GetFolderPath('Desktop')` 取真實桌面

### 問題 11:視覺增強模式按開始評審沒反應
- **狀況**:OpenRouter Nemotron 9B 不支援多模態,選了視覺模式但模型讀不了
- **解法**:Sidebar 開啟視覺時自動過濾出 vision-capable 模型 + 紅色錯誤提示
